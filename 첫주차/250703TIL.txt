        float p = Random.Range(0, 10);
        if (p < 2) Instantiate(normalCat);

-> 이게 왜 20%인가?

💡 왜 20%냐면?
Random.Range(0, 10) // → 0 이상 10 미만의 float을 반환
예를 들어 나올 수 있는 값은:

0.0 ~ 9.9999... 까지
그 중 p < 2이면:

0.0 ~ 1.9999… → 전체 범위의 2/10 = 0.2

즉, 20% 확률로 Instantiate(normalCat)이 실행되는 거예요.




- 특강

대소문자 표기 차이
GameObject - gameObject
Transform - transform

객체 지향 프로그래밍
데이터와 행위를 통해 상호작용 하는 플밍


클래스 같은 종류의 집단에 속하는 데이터, 행위 정의 -> 스크립트 파일하나 = 클래스
클래스의 인스턴스 - 설계도는 같으나 독립적으로 영향
클래스의 인스턴스 사이의 상호작용을 통해 작동

게임오브젝트는 유니티 씬 내 모든 개체의 기본단위
독자적으로 하는 것 보단 실직적기능을 수행하는 컴포넌트 컨테이너 역할을 함
-컴포넌트 클래스를 상속받는 클래스

하이어라키 안에있는 오브젝트들이 GameObject클래스의 인스턴스들
인스펙터는 현재 선택된 게임오브젝트에 부착된 컴포넌트(의 인스턴스)

모든 컴포넌트는 반드시 하나의 게임 오브젝트 인스턴스에 부착되어있음.
소문자 게임오브젝트는 해당 컴포넌스의 부착된 게임 오브젝트 인스턴스에 접근 - 프로퍼티 사용인데 뭔소리냐
아 하이어라키에서 붙여놓은 객체 gameObject라는 내용인듯
그 스크립트가 들어가있는 파일
대문자 게임오브젝트는 걍 클래스라고 보면 된다.


.GetComponent = 갖고있는 컴포넌트중에 하나 갖고오겠다는 소리
인스펙터에 있는 항목들은 모두 스크립트에서 제어가능

대문자 트랜스폼 - 게임 오브젝트 인스턴스가 공통적으로 갖는 기본적인 데이터 관리하는 컴포넌트 (위치 크기등 부모자식 관계포함)
모든 게임오브젝트는 트랜스폼 컴포넌트 인스턴스를 무조건 갖고 있음

소문자 트랜스폼 - 게임오브젝트.transform
붙어있는 애들의 트랜스폼을 표시하는 것. 소문자 게임오브젝트랑 비슷함

게임오브젝트, 트랜스폼앞에 . 이 안 붙는 경우는 this가 생략된 경우
